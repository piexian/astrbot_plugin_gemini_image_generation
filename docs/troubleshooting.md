# 故障排除

## 常见问题

| 问题 | 处理方式 |
|------|----------|
| API 错误 | 检查 API Key、模型名称、`api_type`、`provider_id` 是否匹配 |
| 生成超时 | 降低分辨率、简化提示词、增加工具超时时间，推荐 `100s+` |
| LLM 工具生图后报 `413` 或上下文爆炸 | 升级到 `v1.9.12+`，该版本已过滤 `thought_signature` |
| 无法获取头像 | 确认使用 NapCat 平台，并检查网络权限 |
| 切图效果不佳 | 尝试 `/切图 4 4` 手动指定网格，或配置 `vision_provider_id` |
| 白底透明化不稳定 | 优先使用主体吸附模式；复杂背景建议用专门抠图工具处理 |
| 中文乱码（local 模式） | 等待字体自动下载，或手动放置 `.ttf` 字体到 `tl/` 目录 |
| 网络连接失败 | 在 `api_settings.proxy` 或 provider override 中配置代理地址 |
| OpenAI Images 尺寸报错 | 检查 `custom_size` 是否为合法 `WxH`，并满足官方尺寸约束 |

## OpenAI Images 尺寸检查

`size_mode=custom` 时，`custom_size` 必须满足：

- 格式为 `WxH`，支持 `x` 或 `×`
- 最大边 `<= 3840`
- 宽高均为 `16` 的倍数
- 长短边比 `<= 3:1`
- 总像素在 `655360-8294400` 之间

示例：

```text
1024x1024
1536x1024
2048x1152
3840x2160
```

非法示例：

```text
2048x1080  # 高度不是 16 的倍数
4096x2304  # 最大边超过 3840
2048x512   # 长短边比超过 3:1
```

## 表情包切图限制

自动切图不是稳定的排版引擎，效果取决于模型输出：

- 模型没生成稳定网格时，网格检测会失败或切偏。
- 文字、装饰、边框可能被主体吸附算法识别为独立对象。
- 白底透明化依赖前景/背景差异，淡色描边、复杂背景、阴影会降低成功率。

建议按顺序尝试：

1. 使用 `/切图 4 4` 明确指定网格。
2. 在提示词里要求统一边距、规则网格、不要贴边。
3. 配置 `vision_provider_id` 让视觉模型辅助识别行列。
4. 对特别重要的表情包使用专门切图或抠图工具。

## 调试方式

开启 AstrBot debug 模式查看详细日志。切图相关日志会记录网格识别、主体吸附、远程图片下载和裁剪耗时。

如果需要反馈问题，请尽量附上：

- 插件版本
- AstrBot 版本
- API 类型和模型名
- 触发命令
- 相关日志
- 原图或生成图
